Godot Port — Parity Gap Review

A feature-by-feature audit of the godot/ port against the React+Phaser source (src/), grounded in live play-testing of the running Godot build. The headline finding is the farm tile selector; the rest is a prioritised task list of every functional gap found.

Reviewer: Claude · 2026-06-08 · Godot 4.6.2 build, mobile-portrait 720×1280 · scope: gameplay logic & player-facing behaviour, not cosmetics

32 React slices live-tested 32 tasks 7 bugs · 16 missing · 9 parked

Overview

The Godot port is far more complete than "barebones." The Town 1→2→3 slice plays end-to-end, every nav surface renders cleanly, and most progression systems (quests, almanac, achievements, daily rewards, story engine, tutorial, NPC bonds, workers, castle, decorations, portal, charter) are faithful logic ports. The gaps cluster in a handful of deep systems.

TL;DR. The user-reported bug is real and it is the #1 priority: the Godot "Start Farming" picker lets you toggle whole tile categories on/off, but the original locks the home categories ON and each slot is a picker for which specific tile variant (Grass vs Meadow vs Spiky…) is active. Behind that single screen sits an entire missing subsystem — active-tile-per-category + tile discovery/research/buy — which is also why the 79-tile collection is a read-only gallery you can't actually use.

How this was tested

I ran the real Godot 4.6.2 binary and captured every nav surface from a fresh save and a granted (City-tier, all-buildings) save, then read the pixels myself. I also started live bounded farm runs and dumped active_tile_pool() to verify board behaviour, and ran three parallel read-only code comparisons across all 32 React feature slices vs their Godot equivalents. Findings below cite real file:line on both sides.

Live-verified, not matrix-trusted. Two things I confirmed by running the game that a code skim would miss: (1) selecting only [grass, grain] in Start Farming does not restrict the board — the dumped pool still contained all six category tiles, the selection only appends a +6 weight boost; (2) the Craft (TownScreen) renders fully once the town map is hidden, proving the empty-looking deep-link capture is the nav/layering issue in T28, not a broken screen.

★ The farm tile selector (the reported bug)

This is one screen with four distinct defects stacked behind it. The screen is godot/scenes/StartFarmingModal.gd, claiming to mirror src/features/zones/StartFarmingModal.tsx.

T1 · Replace category toggles with per-category variant pickers [high · bug]

React: when a zone exposes ≤8 categories (mustPick === false, home = 6), every slot is locked ON (StartFarmingModal.tsx:286-289,354-363) and tapping a slot opens TileChooserPopup to pick a variant within that category. Godot builds toggle_mode=true buttons that turn the category on/off (StartFarmingModal.gd:184-196,283-288); there is no variant picker and no "✎ change" affordance. Rebuild the modal as a locked-on grid of variant slots with a chooser popup.

T2 · Add active-tile-per-category state + board substitution [high · missing]

The whole substrate is absent. React keeps state.tileCollection.activeByCategory (category→tileId), set by SET_ACTIVE_TILE (src/state.ts:1349-1369); the board pool substitutes the active variant for the category's base key (tileCollection/effects.ts getActivePool) and upgrade-tile resolution honours it (zones/data.ts nextResourceForZone:303-310). Godot has no activeByCategory, no SET_ACTIVE_TILE; the pool hard-codes one representative tile per category via FARM_CATEGORY_TILE (GameState.gd:1598-1605). Add the state, the setter, and wire it into active_tile_pool() + upgrade_spawn.

T3 · Tile discovery / research / buy economy [high · missing]

React unlocks variants via five methods — default, chain, research, buy (BUY_TILE), building, daily (tileCollection/effects.ts, state.ts:191-218,1396-1413). None exists in Godot — TileCollectionScreen.gd:13-15 states discovery/research/buy "is a LATER milestone … NOT ported," so the gallery is read-only. Without this, the entire variant-unlock progression (and the point of T1/T2's chooser) is missing.

T1–T3 are one feature. Shipping the picker (T1) without the active-tile state (T2) and an unlock economy (T3) just renders a chooser with one option per category. Treat them as a single milestone, smallest-first: T2 state → T1 UI (chooser over discovered variants, default-discovered set seeded) → T3 unlock methods.

T4 · Tile categorisation data bug: Clover & Melon mis-filed under Birds [med · bug]

Visible in the Tile Collection: "Clover" and "Melon" appear under Birds producing eggs. But React authors them category:"flowers" and category:"fruits" (tileCollection/data.ts:126,135) — they only carry a legacy tile_bird_ id prefix. Godot took the prefix at face value: CATEGORY[BIRD_CLOVER]="birds", produces "eggs" (Constants.gd:610-611,339-340). Re-file by authored category and fix the produced resource; the collection groups by Constants.category_of(), so fix it there.

T5 · Per-tile abilities not ported [med · missing]

React variants carry abilities expanded into effects (data.ts:686-694): Turkey/Clover/Melon/Palm grant free_moves, Golden Apple/Coin grant coin bonuses, Pig grants free_turn_if_chain; the freeMoves accumulator is consumed in the reducer and shown in the HUD. Godot has no tile-ability/effects system (TileCollectionScreen.gd:13). This is what makes choosing a variant matter — without it, variants are cosmetic.

T6 · "Off" categories still spawn (soft boost ≠ hard filter) [med · bug]

Even taking the toggle at face value, it barely works. GameState.gd:1713-1723 documents the "DOCUMENTED DIVERGENCE from React: there selectedTiles is a hard FILTER … the port reframes it as a soft weight BOOST." I confirmed live: start_farm_run(["grass","grain"]) still produced a pool containing tile ids 0,1,6,2,3,4 (all six categories) — only +6 grass/wheat were appended. So dropping a category does almost nothing. Resolving T1 (variant pick, no drop) makes this moot; if any drop affordance survives, make it a real filter.

Hazards

React farm hazards = rats, fire, wolves; React mine hazards = cave_in, gas_vent, lava, mole. Godot ships only rats (farm) and rubble (mine), and the rats it has behave differently.

T7 · Fire [high · missing]

React: spawns 4%/fill, spreads 50%/turn to adjacent cells burning resources (cap 3), extinguished by chaining fire tiles for +2 coins each (farm/hazards.ts:40-167,232-251). No fire in godot/ at all.

T8 · Wolves [high · missing]

React: spawn 6%/fill when eggs>30 or turkey>5, eat adjacent bird tiles each turn, scared 5 turns by a Hound or cleared by a Rifle (hazards.ts:43-49,102-116,174-224). No wolf logic in Godot.

T9 · Rats are inert, not pests [high · bug]

React rats tick and eat an adjacent plant tile/turn, spawn on grass>50 & wheat>50, and clear by chaining 3+ rats → +5 coins each (farm/rats.ts). Godot rats are inert pool tiles producing nothing, seeded flat when Town 2 is done, cleared only by Ratcatcher buildings (Constants.gd:284-287, GameState.gd:1727-1729). Port the real lifecycle.

T10 · Rat-interaction tile tags [med · missing]

attracts_rats (Manna/Jackfruit), deadly_pests (Cypress/Beet/Phoenix exterminate adjacent rats), avoids_rats (not eaten) — no tag system in Godot. Coupled to T5/T9.

T11 · Mine hazards 4→1 [med · parked]

React's cave_in / gas_vent / lava / mole (weighted roll, timers, spreading, row-burial) reduce to one static rubble tile cleared by a stone chain (mine/hazards.ts vs Constants.gd:51-58). GameState.gd:85 defers mine hazards "to a later milestone."

Seasons, fertilizer & tools

Seasons — at parity (no task) [verified]

Contrary to an older note, per-season tile-drop weights do differ per season: active_tile_pool() reads ZoneConfig.season_drops() (GameState.gd:1686-1746), a verbatim port of React's seasonDrops. The season bar is functional. One minor caveat (T13).

T12 · Fertilizer entry-item & turn-budget bonuses stubbed [med · missing]

React's modal offers "Use Fertilizer — doubles turns" consuming state.tools.fertilizer, and turnBudgetForZone adds worker/building turn bonuses + tools.extraTurn (zones/data.ts:173-196). Godot keeps the ×2 formula tested but _has_fertilizer() is always false (GameState.gd:1328-1333) so the checkbox is omitted, and the budget is base-turns-only (no building/worker extension). The fill-bias tool path is wired; the run-entry item is not.

T13 · SEASON_POOL_MODS additive deltas not ported [low · missing]

React applies a second seasonal layer on top of seasonDrops: SEASON_POOL_MODS (Spring +1 blackberry, Autumn +2 oak, Winter +1 stone −1 grass) via applySeasonPoolMods (constants.ts:1123-1128, poolMath.ts:16-31). Godot relies solely on the percentage table. Minor spawn-frequency divergence.

T14 · Hazard/utility tool gaps [med · missing/parked]

The Godot catalog is broad (40 wired) but misses: scatter_hazard (timed 5-turn scare / Hound — no timed-scare primitive), water_pump (lava→rubble), explosives (mole + cave-in), and the deferred undo_move (Hourglass, needs a board snapshot) + reveal_tiles (Miner's Hat, needs a hidden-tile layer). Most are blocked on the missing hazards (T7/T8/T11), so they land with those mechanics.

Crafting, market & buildings

T15 · Crafting catalog ~2 of ~46; no Workshop / tool-crafting [high · missing]

Godot ships exactly two recipes — bread, supplies (RecipeConfig.gd:32-52) — across two stations (Bakery, Kitchen). React has ~46 recipes across 5 stations (constants.ts:905-965): ~30 Workshop tool recipes (rake, axe, sickle, rifle, drill, gold_pick…), Bakery goods, Larder, Forge, Kitchen, Smokehouse. The port has no Workshop, so there is no in-game tool crafting at all — tools only arrive via daily-reward grants / portal summons. Recipe tier/level gating is also dropped.

T16 · Market: no drift, no events, no tile-selling [high · missing]

React's market drifts every price ±15%/season and rolls seasonal events (Wood Shortage ×2, Bumper Crop ×0.5, Iron Rush ×2.5…), re-rolled on CLOSE_SEASON (src/market.ts). Godot uses flat static price tables (MarketConfig.gd:18-68) — no drift, no events, no seasonal re-roll (explicitly listed un-ported in GameState.gd:1342). React also sells raw board tiles + fish goods (~36 goods); Godot sells ~18 bare resources, no tile keys. The whole price-timing layer is gone.

T17 · Buildings: ~30 ability-bearing landmarks → 7 spawner/refiner/hazard [high · missing]

React BUILDINGS (constants.ts:760-903) is a ~30-entry catalog of landmarks, many carrying gameplay abilities (turn_budget_bonus, inventory_cap_bonus, bonus_yield, season_bonus, grant_tool, preserve_board, threshold_reduce). Godot BuildingConfig.gd:65-140 is a 7-building model: 3 spawners, 2 refiners, 2 hazard buildings. None of React's ability-bearing buildings exist — no Granary cap, Sawmill/Stable yield, Chapel coins, Powder Store tools, Silo/Barn board-preserve, Housing worker-pool, Observatory threshold. The building→economy effect layer is essentially un-ported.

NPCs, orders, workers & settlements

T18 · NPC gift system absent [med · missing]

React NPCs accept gifts with tiered preferences (loves/likes → +0.5/+0.3/+0.15 bond, once-per-season cooldown, GIVE_GIFT — npcs/data.ts:20-26, bond.ts, state.ts:1415-1431). Godot has no gift mechanic, no preferences, no cooldown; bonds move only via order fills (+0.3) and season decay. A whole interaction loop is removed. (Bond bands/decay themselves are a faithful port — not a gap.)

T19 · Orders: flat reward, no crafted orders, no level scaling [med · missing]

Godot reward = sell_price × qty × 3 then bond-adjusted (OrderConfig.gd:24-31). React uses value-scaled curves (resource max(20, need×value×6), crafted need×value×1.5), a 30%-chance crafted-good order pool (level 3+), and level-scaled quantities (state/helpers.ts:180-290). Godot orders only ever request raw resources at flat qty, and generate_order can duplicate an NPC/resource across the 3 slots (no exclusion, GameState.gd:1050-1070).

T20 · "Workers" tab shows NPC bonds, not the hire system [low · bug]

The Townsfolk screen's Workers tab renders the NPC roster + bond bars (TownsfolkScreen.gd:174-182), while the actual hire-by-type worker system (Farmer/Lumberjack/Miner/Baker) lives on the Craft/TownScreen. React keeps NPCs and hired workers as distinct concepts. A naming/feature collision that will confuse players looking for hiring. (The hire system itself is wired — not a gap.)

T21 · Worker/building/tile ability-aggregation engine missing [med · parked]

React folds workers + buildings + active tiles into one aggregator (workers/aggregate.ts:135-167) feeding many channels (bonusYield, seasonBonus, hazardSpawnReduce, hazardCoinMultiplier, chainRedirect, poolWeight, freeMoves, seasonEndTools, boardPreserve). Godot implements only two worker channels (threshold-reduce-category, recipe-input-reduce). The passive-ability layer is the connective tissue for T5/T12/T17 — worth building once those land.

T22 · Multi-settlement founding + per-zone inventory [high · parked]

React keys inventory & resourceProgress per settlement zone (state/zoneInventory.ts), with a founder-picker and independent stockpiles. Godot uses a single shared global inventory and one Settlement climbing one tier ladder (GameState.gd:80-92) — no founding, no per-zone buckets, no multi-town. Explicitly deferred by the Direction spec; "Town 2/3" today is the same farm board gaining the mine biome + rats. Large; owner-gated.

Faithful / not gaps (verified). Castle (3 needs + contribute), Decorations (8, correct costs/influence), Portal (8/10 tools live, 2 documented-deferred), Charter (read-only, remapped gates), hire-by-type Workers, NPC bond bands/decay, and the Camp→City tier ladder are all real and wired. The tier ladder is a deliberate port-original redesign (React gates buildings by player level; Godot by tier + plot cost), not a bug.

Expeditions & bosses

T23 · Mine Mysterious-Ore → Rune loop absent [high · missing]

React's signature mine reward: one Mysterious Ore spawns with a 5-turn countdown; chain it with ≥2 Dirt for +1 Rune, else it degrades (mine/mysterious_ore.ts:35-103). Godot has no mysterious-ore primitive at all — the mine's only rune source is gone; runes come solely from the harbor pearl. Not flagged as parked.

T24 · Bosses: 1 of 6, and a different fight entirely [high · missing]

React ships 6 seasonal bosses (bosses/data.ts:23-78) as a timed resource-target challenge (gather N of a resource in a 10-turn window, reward scaled by overshoot) with 6 board modifiers (freeze_columns, respawn_boost, heat_tiles, rubble_blocks, hide_resources, min_chain). Godot has only Frostmaw, as an HP-attrition grind (each chain subtracts its length from HP 40, flat 500-coin reward, no rune/gem/XP), and ports only the min_chain raise (GameState.gd:1794-1824) — even Frostmaw's own freeze_columns:2 is dropped. The capstone framing is intentional, but the mechanic + modifiers are a substantive divergence.

T25 · Fish pearl-capture rule changed [med · bug]

React: the chain must contain the pearl tile + ≥2 other fish (fish/pearl.ts:122-128). Godot ships a faithful try_capture_pearl but the live board uses capture_pearl_if_adjacent (a fish chain ≥2 run 8-adjacent to the pearl, GameState.gd:1559-1577) because the pearl is a non-chainable placed tile. Behaviourally divergent — you never chain the pearl itself. Documented deviation; decide whether to make the pearl chainable to match React, or keep adjacency intentionally.

T26 · Cartography: 11-node travel map → 3-node router [high · parked]

React's MAP_NODES = 11 zones / 14 edges with per-node entry cost, level gates, dangers, distinct board templates, and a real travel state machine (visited/discovered/hidden, fast-travel, adjacency gating) (cartography/data.ts,slice.ts). Godot has 3 nodes (home/mine/harbor), 2 edges, no travel mechanic — the screen just routes a tap into enter_mine/enter_harbor (CartographyConfig.gd:28-48). Explicitly a trimmed "flavor view"; tied to T22.

T27 · Fish achievements omitted on a stale reason [low · bug]

AchievementConfig.gd:45-46 drops first_catch/tide_runner/master_angler "because the port has NO harbor / fish biome" — but the port does have a working harbor. The three trophies are reachable; they're unreachable only because fish_chained isn't bumped at credit_chain. Cheap fix: wire the counter, re-add the trophies.

Meta, story & UX

T28 · Deep-link to a primary view leaves the town map on top [low · bug]

The in-game nav tabs route through _switch_primary_view (hides sibling views), but the deep-link path (apply_deeplink) and direct opens do not. I verified: after _open_town(), both _townmap_screen.visible and _town_screen.visible are true, and the map (higher canvas layer) renders over the Craft screen. Affects hash deep-links and browser back/forward on the web export, and any QA capture via deep-link. Fix: have apply_deeplink for the 5 primary views go through _switch_primary_view (or hide the map) like the nav tabs do.

T29 · Story/chronicle content trimmed [med · missing]

React = 15 main + 10 side beats, ~29 flags; Godot = 14 beats, 15 flags, with branching reduced to one beat (frostmaw_aftermath) (StoryConfig.gd). The engine/condition-evaluator/choice machinery is a faithful port; omissions are gated on absent systems (NPC arrivals, bonds, keepers, festivals, the mira_letter arc). Will fill in as T18/T22/etc. land.

T30 · Run summary simplified [low · missing]

React runSummary tracks biggest-chain, bond deltas, beats-triggered, per-resource gains, supplies consumed. Godot's HarvestModal recaps the season + a turn/economy snapshot + return bonus only (HarvestModal.gd). Also: close_season deliberately omits market drift, the session_ended beat, season-end tool grants, board-preserve, and freeMoves reset (GameState.gd:1340-1348).

T31 · Boons absent; achievements/almanac deltas [low · parked]

The Boons slice (ember/coreIngot purchases) has no Godot equivalent; Festivals are stubbed both sides (parity). Achievements roster is 27→19 (omissions unreachable in the port); the four Almanac "structural perks" (extraTurn, goldSeal…) are inert flags — but they're inert in React too, so this is parity, not regression.

T32 · Verify season bar renders during a live run [low · verify]

My synthetic board capture (run started via start_farm_run directly) did not visibly show the season strip; the season logic is wired, so this is most likely a HUD-refresh timing artifact in my capture rather than a real bug. Quick check: start a run via the in-game Start-Farming flow and confirm SeasonBar is visible.

Suggested sequencing

1 · The reported bug (one milestone)

T2 active-tile state → T1 variant-picker UI → T3 unlock economy, plus the cheap data fix T4. This is what the user is looking at; do it first and whole.

2 · Quick correctness wins

T27 (wire one counter), T28 (deep-link hides map), T20 (Workers-tab label), T32 (verify). Small, isolated, high signal-to-effort.

3 · Farm depth

Hazards T9/T7/T8 + their tools T14; then T5 tile abilities + T23 mysterious ore. Makes the board interesting again.

4 · Economy & content

T15 recipes/Workshop, T16 market, T17 buildings, T19/T18 social. Larger; sequence behind the ability engine T21.

5 · Owner-gated / parked

T22 multi-settlement, T26 full world map, T24 boss roster/mechanic. Big scope; needs a design call before building.

Note on "no-fakes." The port deliberately omits UI for systems it hasn't ported (to avoid fake controls). Each task above should port the real mechanic end-to-end, not a stub — consistent with the repo's engineering rules.
