# Hearth tile — style reference

This is the **prose half** of the cohesion anchor for the sprite pipeline. It reads the
machine-readable contract in [`_style-spec.json`](./_style-spec.json) and the visual sheet in
[`_style-reference.png`](./_style-reference.png) into plain English — the rules a generated tile
must obey to look like it belongs to this set. Every generated still and animation is scored
against the spec at the **G1/G2** critique gates, so this doc and the spec are the same contract,
told two ways.

> Generated by the `style-reference` skill from `.claude/skills/style-reference/assets/manifest.hearth.json`.
> Numbers below are transcribed from `_style-spec.json` — **edit the spec, not invented values here.**

## The three artifacts (and which is authoritative)

| Artifact | Role | Authoritative for |
|----------|------|-------------------|
| `_style-spec.json` | machine contract — the cohesion anchor | **everything**: palette ramps, light, outline, shadow, perspective, animation defaults |
| `_style-reference.png` | the visual sheet (exemplars + palette swatches + labels) | a human glance — "does new art look like this?" |
| `_style-reference.md` *(this file)* | prose reading of the spec | onboarding / review checklist; defers to the spec on any number |

When the spec and this doc disagree, **the spec wins** — regenerate this doc from it.

## The look at a glance

Glossy, rounded, three-quarter top-down game tiles with a clear single light source, hue-shifted
color ramps (cool shadows, warm highlights), a selective dark outline, and a soft drop shadow that
grounds each object. The hero exemplars on the sheet are the shipped `oak`, `fir`, `squash`, and
`carrot` tiles; the birch `autumn`/`winter` pair and the autumn idle frame-strip show how a *family*
(seasonal keyframes + their animation) sits inside the same look.

## Canvas

- **Reference resolution:** `90×90` px (the source art the look was authored at).
- **Safe area:** `6` px inset kept clear of the edge so nothing important clips on the board.
- **Background:** `transparent` — tiles composite over the board.
- **Build size note:** the pipeline's actual output is **`32×32`** (`pipeline.json` `settings.canvas`),
  which **supersedes** the spec's reference `canvas` for the build target. The `90×90` here records
  what the look was drawn at, not what ships.

## Palette — the #1 cohesion lever

Six **locked, named ramps**, each `darkest → brightest`, taken from `_style-spec.json`
`palette.ramps[]` (`palette.source` names `references/palette.gpl` as provenance):

| Ramp | Hexes (dark → light) |
|------|----------------------|
| `foliage-green` | `#1a3008` · `#2a5010` · `#5c9c2e` · `#98c171` · `#a8d858` |
| `bark-brown` | `#2a1c0c` · `#543110` · `#744617` · `#9a6c34` · `#c79a5a` |
| `soil-earth` | `#2c1c10` · `#4e3413` · `#6f4a1f` · `#9a6c3a` · `#c49a63` |
| `wheat-gold` | `#5a3a10` · `#8a6418` · `#c79a2e` · `#e3c45a` · `#f4e08a` |
| `pumpkin-orange` | `#4a1f10` · `#8a3c12` · `#c3671d` · `#e8923c` · `#f4b95e` |
| `stone-gray` | `#1b1c1a` · `#3a3c38` · `#5c5f58` · `#888c82` · `#c2c6ba` |

The ramps are **hue-shifted**, not fixed-hue value ramps — this is what reads as crafted rather than
plastic:

- **Shadow hue shift:** `−20°` (cool/blue-shifted at the dark end).
- **Highlight hue shift:** `+20°` (warm/yellow-shifted at the bright end).

> The swatch row on `_style-reference.png` is **quantized live from the exemplar pixels** (the
> manifest used `"palette": "auto"`), so it shows the colors actually present in the sample art.
> The **locked** ramps above — the ones generation targets and G2 scores against — are the table in
> `_style-spec.json`. Point a manifest's `"palette"` at a `.gpl`/`.hex` file to put the locked ramps
> on the sheet verbatim instead.

**Extending the palette for a new material** (the sanctioned move): anchor a new ramp's midtone to a
real sampled hex from the closest shipped sibling, hue-shift the dark/light steps by the rules above,
then **append the new named ramp to `palette.ramps[]` and lock it** before generating. The
`pumpkin-orange` ramp is the worked example — sampled from `squash`/`carrot`, hue-shifted, then
locked.

## Light

- **Direction:** `upper-left`, held across the whole set.
- **Elevation:** `55°` (governs shadow length and top-face brightness).
- **Contrast:** `medium`.

## Outline

- **Rule:** `selective` — outline only where it reads, not a full keyline.
- **Color:** `#1b1410` (a dark near-black, **not** pure `#000000`).

## Shadow

- **Type:** `soft-drop` (a blurred cast shadow that grounds the object).
- **Color:** `#0000003b` (black at ~23% alpha).
- **Offset:** `x: 2`, `y: 3` px.
- **Blur:** `2` px.

## Framing & texture

- **Perspective:** `three-quarter-topdown` (the board's default camera).
- **Dither policy:** `minimal` — dithering allowed only sparingly for gradients/texture.

## Animation defaults

- **FPS:** `10` (reference/default; `pipeline.json` `settings.fps` and any per-item `fps` override
  win for the actual build playback rate).
- **Default frame count:** `8` for an idle when an animation omits `frames`.
- **Cadence:** `on-twos` (hold each drawn frame two display frames).
- **Loop:** `true` (idles loop seamlessly).
- **Idle tag name:** `idle` (also the SpriteFrames default tag).

The autumn idle frame-strip on the sheet (frames 0, 2, 4, 6 of the 8-frame birch idle) shows the
motion model: the tree holds rigid while a single gold leaf detaches and flutters down once per loop.

## References (what the spec was derived from)

`_style-spec.json` records `references[]` = `../tile_tree_oak.png`, `../tile_tree_fir.png`,

## How this enforces consistency (the short answer)

1. **The locked palette ramps are the strongest single lever** — one shared ramp set makes disparate
   subjects read as one family even when silhouettes differ.
2. **The sheet + spec are the single contract** both generation targets and the G1/G2 gates score
   against. Off-palette color, wrong light direction, or a missing outline rule is a *reject*, not a
   style choice.
3. **New materials extend the palette via the sanctioned ramp-extension move** (sample a sibling,
   hue-shift, lock) — so growth stays cohesive instead of drifting.
